---
title: "Content Ownership"
description: "The boundary between what content repos control and what the docs system controls."
---
The docs system only stays stable if content and rendering have a hard boundary.

## Content Repo Owns

- document text
- metadata
- navigation order
- page type
- structured data for cards, items, sections, and links

## System Repo Owns

- shell layout
- page templates
- card layouts
- typography rules
- docs blocks
- preview/code composition
- deploy pipeline

## What Should Not Happen

Content repos should not be able to change:

- matrix vs list layout
- card spacing
- code block chrome
- shell-level visual behavior

If a content change can alter those things, the system boundary is still weak.

## Product Implication

This is why the docs system should evolve toward:

- schema-driven content
- template-driven rendering
- CMS editing over structured data

not freeform MDX page composition for every important page.
