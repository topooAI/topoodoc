The content schema defines what content repos are allowed to describe.

It does **not** define final visuals.

## Page Types

The first stable page types are:

- `doc-page`
- `board-landing`
- `component-index`
- `docs-system-index`
- `docs-block-doc`

## Why Schema Exists

Even with a separate content repo, docs content can still break layout if it stores:

- raw layout wrappers
- class names
- visual composition logic
- grid decisions

Schema exists to stop that drift.

## Current Rule

Content repos may define:

- text
- links
- data structure
- section order
- page metadata

Content repos may not define:

- shell layout
- page-wide visual structure
- card rendering rules
- typography system
- block renderer implementation

## Current Source Of Truth

The current formal schema contract lives in:

- [CONTENT_SCHEMA.md](https://github.com/topooAI/topoodoc/blob/main/docs/CONTENT_SCHEMA.md)

That contract is owned by the system repo, not the content repo.
