Topoo docs need their own UI blocks.

These are not generic product components. They are documentation rendering primitives.

## Current Block Direction

The first important docs blocks are:

- `component-preview-block`
- `preview-code-block`
- `installation-block`
- `package-tabs-block`
- `callout-block`
- `steps-block`

## Example Pattern

One of the key docs patterns is:

- preview on top
- code below
- optional `View Code` interaction

That pattern should be modeled as a docs block, not hand-composed in content pages each time.

## Ownership

Content owns:

- which block is used
- which demo/source ids are referenced
- explanatory text

System owns:

- preview shell
- code shell
- toggles
- spacing
- typography
- interaction behavior
