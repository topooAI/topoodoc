---
title: "Schema"
description: "The content model used by the Topoo documentation system."
---
The documentation system separates **content structure** from **rendering structure**.

Schema defines:

- which page types exist
- which block types exist
- what fields each type is allowed to carry
- which parts are content-owned versus system-owned

In practice, schema is the contract between:

- pure content repositories
- the Topoo docs rendering system
- future CMS or publishing workflows

## Current Page Types

- `doc-page`: long-form narrative documentation
- `board-landing`: a board-level landing page
- `component-index`: a component reference index

## Current Rule

If a change only affects text, metadata, links, or ordering, it can live in a pure content repo.

If a change can affect layout, typography, spacing, card structure, or renderer composition, it belongs to the system layer.

## Why This Matters

Without schema, content and layout get mixed together.

That is exactly what causes a content publish to accidentally change the visual output of the docs site.
