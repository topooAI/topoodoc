title: "Blocks"
description: "The documentation primitives used to render structured docs content."
---
Topoo docs rely on a small set of documentation-specific blocks.

These are not generic product UI components.

They are **docs rendering primitives**.

## Current Docs Blocks

- prose sections
- linked cards
- callouts
- steps
- tabs
- package command tabs
- preview and code pairs

## Ownership Rule

The visual behavior of docs blocks belongs to the system layer.

The content layer may provide:

- text
- code samples
- labels
- metadata

The content layer should not redefine how these blocks are laid out or styled.

## Why Blocks Exist

Blocks give the docs site a repeatable rendering language.

They are the bridge between schema and the final page UI.
